Lets read about SOLID

SOLID is an acronym that stands for five object-oriented design principles that help you make the software more understandable, easier to maintain, and easier to extend.Below are the examples from Calculator app about sold principles to understand by examples.

  • Single-responsibility: As the name suggests, single responsibility means a call should have only one job. In our calculator app, in the operation and calculation names, you can see that we have separate classes for each operation or calculation, which shows the single responsibility principle in the calculator app.
    class Division:
    """This is the Division class"""

    @staticmethod
    def divide(value_1, value_2):
        """ This is the Divide method"""
        try:
            return value_1 / value_2
        except ZeroDivisionError:
            return "you can't divide by zero"


class Multiplication:
    """ This is the Multiplication class"""

    @staticmethod
    def multiply(value_1, value_2):
        """ This is the add method"""
        return value_1 * value_2


class Addition:
    """ This is the addition class"""

    @staticmethod
    def add(value_1, value_2):
        """ This is the add method"""
        return value_1 + value_2


class Subtraction:
    """ This is the subtraction class"""

    @staticmethod
    def subtract(value_1, value_2):
        """ This is the add method"""
        return value_1 - value_2
  • Open-closed: This principle tells us function should open for extension but close for modifications. so in the calculator class we are coding to a interface named calculations for specific class and also in the calculations interface we are importing another interface called operations and all the operation will perform there. so basically Coding to an interface is an integral part of SOLID so that's how open-closed principle used in calculator app we are extending our interfaces and performing operations and those operations methods are closed for any modifications they just take values and return the results.

    in below code get_result() method will get called from interface calculations so we can extend or do changes in the Division class in calculations interface but we can't do any change in Division class from operations interface.

    class Division(Calculation):
    """Division calculation object"""

    def get_result(self):
        """get the Division results"""
        result = 1.0
        for value in self.values:
            result = Div.divide(result, value)
        return result
